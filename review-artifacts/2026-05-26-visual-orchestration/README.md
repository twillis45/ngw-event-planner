# Sprint 22 — Visual Orchestration Language

The highest-leverage layer in the entire product. The system already thinks correctly. Now it sees correctly.

## What this is

This is NOT analytics expansion. This is NOT dashboard design. This is the creation of a visual orchestration language — spatial, temporal, and relational cognition infrastructure that communicates orchestration pressure without charts, metrics, or dashboards.

## Critical philosophy

The planner should feel: "I understand the SHAPE of this event."

NOT: "I am reviewing metrics."

The UI feels like backstage orchestration, luxury hospitality coordination, architectural sequencing, cinematic timing awareness, institutional event memory.

## Visual orchestration primitives introduced

- **Sequence Ribbons** — horizontal bands where width = duration, spacing = breathing room, left-border accent = vendor color
- **Timing Corridors** — temporal zones showing prep→event flow with opacity gradient
- **Dependency Bands** — connected ribbon corridors where handoff points are visible seams
- **Pressure Clustering** — multiple ribbons stacking vertically to show orchestration density
- **Day Rhythm Strip** — compressed horizontal band showing the event's tempo: setup peak, rehearsal freeze, ceremony, transition, reception
- **Compression Zones** — highlighted regions where multiple vendors compete for time/space
- **Lane Narrowing** — ribbon width proportional to time available; narrow = fragile
- **Fragile Transitions** — amber-bordered narrow ribbons showing zero-margin handoffs

## Pressure visualization mechanisms (NO charts)

| Mechanism | What it communicates | How |
|---|---|---|
| Spacing compression | Temporal pressure | Elements crowd together as time shrinks |
| Lane narrowing | Available margin | Ribbon width IS the temporal margin |
| Rhythm shifts | Tempo change | Irregular spacing = pressure accumulation |
| Typography pressure | Urgency gradient | Regular → Medium → Semi Bold → Bold |
| Layering density | Coordination load | Stacked ribbons = overlap = congestion |
| Temporal clustering | Pressure zones | Activities grouped by time proximity |

## Mobile orchestration compression

Mobile surfaces are NOT stacked desktop. Each mobile surface uses orchestration-native cognition:

- **Surface 08 (Compressed Review)**: Day rhythm scan strip + glance-state chips + folded outstanding cards + compressed sequence
- **Surface 09 (Temporal Scan)**: NOW/NEXT/AFTER progressive reveal — large dominant NOW card, smaller NEXT, compressed rest-of-day
- **Surface 10 (Fragility Sequence)**: Fragile windows as large touchable cards with ACTIVE/UPCOMING/TODAY status + breathing room zones

## Drift risks

1. **Chart drift** — Sequence ribbons could become Gantt charts. They show rhythm, not task management. Width = duration, not progress.
2. **Dashboard drift** — Compression zone cards could become KPI tiles. They show coordination needs, not metrics.
3. **Calendar drift** — Temporal corridors could become scheduling UI. They show orchestration pressure, not available slots.
4. **Analytics drift** — Density strips could become trend lines. They show where vendors overlap, not performance data.
5. **PM drift** — Dependency corridors could become flowcharts. They show handoff fragility, not task dependencies.
6. **CRM drift** — Cross-event corridors could become pipeline views. They show temporal congestion, not deal stages.
7. **Admin drift** — Walking coordination mode could become admin software. It shows NOW/NEXT/AFTER, not settings.

## Enterprise drift analysis

| Pattern | Risk level | Mitigation |
|---|---|---|
| Ribbons → Gantt | HIGH | Ribbons have no progress state. Width = time, not completion. |
| Density strips → sparklines | MEDIUM | Strips show vendor presence, not data points. No axes. |
| Corridors → flowcharts | HIGH | Corridors are spatial, not logical. No decision nodes. |
| Phase bands → timeline UI | MEDIUM | Bands show rhythm, not schedulable blocks. |
| NOW/NEXT → notification cards | LOW | NOW/NEXT shows orchestration context, not alerts. |

## What is NOT validated

- No real planner has reviewed these surfaces. All content is synthetic.
- The visual orchestration primitives (sequence ribbons, dependency corridors) are hypotheses — actual planner spatial cognition may differ.
- Temporal scan mode has not been tested with planners during live events.
- Walking coordination mode touch targets have not been tested on actual tablets.
- The day rhythm strip compression model has not been validated against real event timelines.
- None of these surfaces exist in code yet. They are Figma explorations.

## Honest read

This sprint creates the visual language that was missing. Prior sprints built the cognitive system (what planners think about) and the linguistic system (how the product speaks). This sprint builds the perceptual system — how orchestration pressure becomes visible without falling into chart/dashboard territory. The key innovation is the sequence ribbon pattern: width = duration, spacing = breathing room, color intensity = fragility. This single primitive communicates more about temporal pressure than any dashboard could, because it encodes time spatially rather than numerically. The mobile surfaces break new ground by refusing to stack desktop — the temporal scan mode (NOW/NEXT/AFTER) and fragility sequence are genuinely mobile-native orchestration patterns, not responsive compromises. Whether these visual patterns actually help planners "see the shape of an event" is unknown until field sessions confirm.
