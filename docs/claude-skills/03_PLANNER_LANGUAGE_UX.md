# 03_PLANNER_LANGUAGE_UX.md

You are responsible for making NGW Event Planner speak like an event operations tool.

Use language that event coordinators, planners, and studio operators understand.

## Top-Level Navigation Language

Preferred L1 labels:

- Home
- Events
- Clients
- Schedule
- Studio Setup

Use display labels without breaking route keys.

Examples:
- route key may remain `calendar`, display label becomes Schedule.
- route key may remain `settings`, display label becomes Studio Setup.
- route key may remain `allEvents`, display label becomes Events.

## Event-Level Language

Preferred L4 labels:

- Command
- Vendors
- Decisions
- Timeline
- Checklist
- Communication
- Calendar
- Run of Show
- Budget
- Guests
- Seating
- Manage Event

## Use Planner Urgency Language

Use:

- What needs attention
- Start here
- Next best action
- Blocking event readiness
- Due this week
- Waiting on vendor
- Waiting on client
- Needs confirmation
- Ready for event day
- Not ready yet
- Final count mismatch
- Client approval blocking timeline
- Vendor follow-up needed
- Day-of contact missing

Avoid:

- Items
- Records
- Objects
- Entities
- Miscellaneous
- Pending
- Active
- Normal
- Overview
- Dashboard widgets
- Status data

## Copy Rule

Good UI copy explains consequence.

Bad: Pending

Better: Waiting on signed contract.

Bad: Missing contact

Better: Day-of contact missing — no one to call if setup is delayed.

Bad: Documents missing

Better: Contract is not attached — booking record is incomplete.

Bad: 3 issues

Better: 3 vendor items are blocking event readiness.

Home:

## Rule

Do not make labels cute.

Do not over-brand.

Use clear, familiar, planner-native language.
