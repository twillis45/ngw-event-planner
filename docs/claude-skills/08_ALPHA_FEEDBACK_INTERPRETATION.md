# 08_ALPHA_FEEDBACK_INTERPRETATION.md

You are responsible for interpreting NGW alpha feedback.

Do not treat every comment as equal.

Observed behavior matters more than opinion.

## Strong Evidence

Strong evidence:
- tester cannot start,
- tester clicks wrong surface,
- tester misses primary action,
- tester cannot explain screen purpose,
- tester cannot complete task,
- multiple testers fail same way,
- tester says what current workflow this replaces,
- tester asks when they can use it,
- tester asks about pricing.

## Weak Evidence

Weak evidence:
- "Looks nice."
- "Cool idea."
- "I like it."
- "Could be useful."
- "Maybe later."

## Feedback Translation

If users say: "Uninspiring"
Translate to:
- weak visual hierarchy,
- no emotional contrast,
- no premium moment,
- too generic,
- no category-specific detail.

If users say: "I don't know where to start"
Translate to:
- no dominant starting point,
- no next best action,
- too many equal-weight cards,
- weak command strip,
- poor attention routing.

If users say: "Lacks creativity"
Translate to:
- app does not express event-planning category,
- too much generic software language,
- insufficient planner-specific workflows,
- details do not feel operationally true.

## Response Pattern

Do not respond with decoration first.

Respond with:
1. clearer starting point,
2. better consequence copy,
3. stronger command panel,
4. richer operational detail,
5. planner-native language,
6. runtime improvements.

## Do Not

Do not add:
- generic onboarding modal as crutch,
- confetti,
- random gradients,
- fake AI sparkle,
- cute icons,
- more cards,
- more tabs,
- another dashboard.

Fix the actual product comprehension problem.
