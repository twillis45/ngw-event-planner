## How to run the board
1. **Render-first, always.** Capture screenshots at desktop 1440, widescreen 1728, tablet-land 1024 + portrait 768, mobile 390 → `demo/review-artifacts/`.
2. **Dispatch board agents** (Agent tool, general-purpose) that **Read the PNGs** and cross-reference `file:line`. Code review alone misses emergent "too busy / flat / no-hierarchy" problems that only exist in assembled, data-populated pixels.
3. **Design stars first → synthesize → event-industry stars + Grandmother react and OVERRIDE** where real practice diverges from theory. (Precedent: the pros killed a 3-pane cockpit + 9-cell grid as "the overwhelm we just escaped, repainted.")
4. **Demand BRUTAL TRUTH, explicitly NOT consensus.** Reviewers complete a workflow end-to-end. Each returns ranked findings with `file:line` + an honest /10 score.

**Discipline for culturally/regionally-specific playbooks:** author render-first from insider experts (not stereotype), internal-diversity-aware, ASK-don't-ASSUME, opt-in overlay (never auto-applied by name/inference), anti-appropriation, dignity-first. Quantities grounded in real foodway norms, labeled `verificationStatus:'synthesized'`, `sources:[]` (no fabricated citations) until a foreground verification pass attaches them. Every food playbook clears the Food-Safety seat.

**Why this roster:** collapsing it to generic roles loses the specific lens each star enforces (Tufte = density, Rams/Ive = reduction, Norman = truthfulness, the event pros = lived override authority). This named-star method is the user's actual working practice.
