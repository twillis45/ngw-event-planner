# 05_VENDOR_READINESS_DETAIL_COCKPIT.md

You are responsible for the Vendors workspace and Vendor Detail Cockpit.

The Vendors surface must answer:

Which vendors are safe, which vendors are risky, and what must be done before, during, or after the event?

The Vendor Detail view must win.

## Vendor Detail Is Not A CRM Profile

Bad:
- name
- email
- phone
- status
- amount
- notes

Better:
- readiness
- lifecycle stage
- next best action
- why it matters
- planning readiness
- day-of control
- closeout
- required questions
- linked work
- documents
- notes

## Vendor Lifecycle

Use or map to:

1. Inquiry / Needed
2. Shortlisted
3. Selected
4. Contracting
5. Booked
6. Planning
7. Final Confirmation
8. Day-Of Active
9. Post-Event Closeout
10. Archived

## Vendor Phases

Vendor detail should separate:

- Planning
- Day-Of
- Closeout

## Planning Section

Answers:

What must be true before event day?

Include:
- vendor selected,
- scope confirmed,
- contract status,
- payment/deposit status,
- primary contact,
- day-of contact,
- arrival/setup/strike plan,
- venue/logistics needs,
- required documents,
- required questions,
- linked decisions,
- linked tasks,
- linked timeline items.

## Day-Of Section

Answers:

What do we need when the event is live?

Include:
- expected arrival,
- actual arrival,
- check-in,
- setup status,
- ready status,
- active/complete status,
- strike/load-out,
- day-of contact,
- logistics needs,
- issue notes.

Day-of statuses:
- Expected
- Arrived
- Setting Up
- Ready
- Active
- Complete
- Striking
- Closed

Exception states:
- Late
- Blocked
- Needs Decision
- Needs Payment
- Wrong Location
- Missing Item
- No-Show
- Escalated

## Closeout Section

Answers:

What must happen after the event?

Include:
- final invoice,
- final payment,
- gratuity/tip,
- deliverables received,
- dispute/issue notes,
- thank-you/follow-up,
- vendor rating,
- preferred vendor flag.

## Vendor Challenge Categories

Use:
- Booking
- Communication
- Logistics
- Scope
- Timeline
- Financial
- Documents
- Day-of
- Closeout

Each category:
- Safe
- Needs attention
- Critical
- Not tracked yet

## Vendor Type Questions

Start with templates for:
- Caterer
- Photographer / Videographer
- Florist / Decor
- DJ / Band / Entertainment
- Venue
- Rentals

Unknown answers must show Not tracked yet.

## Detail View Structure

Use:

1. Command Header
2. Why It Matters
3. Readiness Snapshot
4. Planning
5. Day-Of
6. Closeout
7. Required Questions
8. Linked Event Work
9. Documents
10. Notes

## Command Header

Must include:
- vendor name,
- category,
- readiness,
- lifecycle stage,
- next best action,
- consequence explanation,
- primary CTA if safe.

## Missing Data Language

Use:
- Day-of contact missing — no one to call if setup is delayed.
- Contract is not attached — booking record is incomplete.
- Arrival time missing — timeline readiness cannot be trusted.
- Final payment due before vendor is fully secured.

## Do Not

Do not create a duplicate Vendors surface.
Do not fake uploads.
Do not invent vendor facts.
Do not create fake AI output.
Do not make the detail view generic.
